# Section brainstorm from article

Use this command to turn an **article** (or doc) into **atomic, section-template-style goals** for the repo you're in.

**What to do:**

1. **Repo context**  
   Check the current repo: read `IMPLEMENTATION_PLAN.md`, `completed.md`.  Scan the codebase enough to see what’s already implemented and what fits naturally next.

2. **Article + my input**  
   I’ll provide an article (or doc) and my own notes below. Use both to brainstorm.

3. **Output**  
   Propose the **easiest, most atomic** goals that could be added as new sections. For each goal:
   - Use the **section template** structure from the section-documentation-template rule: `## Section N: Title – scope`, **Single goal**, **Details**, **Files and purpose** (table), **How to test**, and optionally **Test results**.
   - Order by: quick wins / least dependency first, then building on those.
   - Skip or merge anything that’s already done (per `completed.md` / repo).
   - Call out which parts need my input (e.g. file paths, tech choices) if something is unclear.

**Constraint:** Only add or edit content in `IMPLEMENTATION_PLAN.md`. Add once I say to. Do not create or write to any other file.

(paste article or doc here)
```

```
